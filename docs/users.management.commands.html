users.management.commands.generate_test_users module

class users.management.commands.generate_test_users.Command(stdout=None, stderr=None, no_color=False, force_color=False)[source]

Bases: BaseCommand

The Command class defines a custom Django management command to generate random users for testing purposes. This command can create either general test users or test demonstrators, with options to specify the number of users.

Parameters:

help (str) -- A brief description of what the command does, which is displayed when the help flag is used.

add_arguments(parser: CommandParser) → None[source]

Define the arguments that the command accepts. The command allows the user to specify the total number of test users to create or the number of test demonstrators to create.

Parameters:

parser (CommandParser) -- The argument parser for the command.

handle(*args, **kwargs)[source]

Handle the execution of the command. This method generates random user data using the Faker library, ensures unique usernames and emails, and creates User objects in the database. If the --demonstrator flag is provided, it ensures that all created users are demonstrators.

Parameters:

  • args (list) -- Additional positional arguments.

  • kwargs (dict) -- A dictionary of options passed to the command, including the number of users or demonstrators.

Returns:

None

help = 'Generates random users for testing purposes'

Bases: BaseCommand

The Command class defines a custom Django management command to generate availability records for users marked as fake. This command is useful for creating test data that simulates user availability across all time slots.

Parameters:

help (str) -- A brief description of what the command does, which is displayed when the help flag is used.

handle(*args, **kwargs)[source]

Handle the execution of the command. This method creates UserAvailability records for each fake user across all available time slots, ensuring that no duplicates are created. It also provides feedback on the number of users for whom availability was created.

Parameters:

  • args (list) -- Additional positional arguments.

  • kwargs (dict) -- A dictionary of options passed to the command.

Returns:

A success message indicating the number of users for whom availability was created.

Return type:

str or None
